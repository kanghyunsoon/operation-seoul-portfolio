## 2. 테이블 설명

| 테이블 | 설명 |
| --- | --- |
| `users` | 사용자 계정, 관리자 권한, 인증 정보 |
| `region` | 작전 지역 카드 단위 정보 |
| `mission` | 지역에 속한 현장 미션과 최종 미션 정보 |
| `game_session` | 사용자별 미션 진행 상태, 인증 로그, 점수 |
| `clear_report` | 최종 클리어 후 역사 해설과 단서 해석 결과 |

## 3. 설계 기준

- 지역 하나는 여러 미션을 가진다.
- 사용자는 미션별로 하나의 진행 세션을 가진다.
- 최종 미션이 클리어되면 `clear_report`가 생성된다.
- 좌표는 지도/API 연동 정확도를 위해 `decimal(10,7)`로 저장한다.
- 외부 API 원문 전체 저장은 MVP 범위에서 제외하고, 필요한 결과만 도메인 컬럼에 저장한다.
